## 机理结论
用 3-5 句给出核心机理判断：是什么因素、通过什么过程、导致了什么结果。

## 因果链解析
沿因果链分环节展开：触发条件/影响因素 → 反应或演化过程 → 中间产物/微观结构 → 最终性能/结果。每个环节用证据支撑，关键论点末尾用方括号引用对应证据 chunk，格式 [chunk_id, section, page N, para M]（page/para 为 1-based，无 para 可省略；chunk_id 照抄证据材料中每条片段方括号里的 chunk_id）；可引用反应式、表征结果、关键参数。

## 关键证据要点表
用 Markdown 表格汇总支撑机理的关键证据，列建议为：环节 | 关键证据/观测 | 来源，其中"来源"列填上述 chunk 引用（[chunk_id, ...] 格式）。

## 机制共识与分歧
指出不同文献对机制解释的一致与分歧之处（若证据不足以判断，明说）。

## 机理缺口与下一步
客观列出因果链上仍缺证据的环节（断点），以及建议补充的检索/研究方向。

硬性要求：
- 严禁脑补因果链上没有证据的环节；缺证据的环节如实标注并列入"## 机理缺口与下一步"；
- 引用一律用证据材料中真实存在的 chunk_id（形如 text_xxx / table_xxx / summary_xxx），严禁编造 chunk_id，也不要只写文献名或纯数字编号；
- 关键论点必须可溯源；反应式/化学式/单位照搬材料写法；
- 证据不足时如实说明，不要臆测填充。
