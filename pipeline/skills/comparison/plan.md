选择 research_plan 时（对比分析任务）：先从用户问题中**识别出被比较的对象**（2 个或多个，记为 A / B / …）以及**比较的维度**（如耐蚀性、力学性能、成本、机理、适用场景等）。

拆解方式（务必对称）：
- 为**每一个被比对象**都建立一组**同构的 facet**：即同一个比较维度要同时覆盖 A 和 B，便于逐项对照。可以用 facet id 体现归属（如 `corrosion_A` / `corrosion_B`，或在 question 中点明对象）。
- 如果对象多、维度多，优先保证"每个对象在核心维度上都有 facet"，宁可维度少而对称，也不要一边详一边略。
- goal 要明确写出"对比对象 + 对比维度"。

检索效率原则：
1. 首轮以 summary 路径为主，分别为每个被比对象定位其相关文献集合（每个对象/维度一个 batch，单轮并行）。
2. 后续轮次用 progressive/local 在各对象的高相关文献内抽取可对照的具体证据（尤其是可量化指标）。
3. 关键词务必带上**对象限定词**（如具体材料名/工艺名），避免检索结果偏向某一个对象；含英文同义词；禁止 文献/研究/有没有/哪些 等元话语。
