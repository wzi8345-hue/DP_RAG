# DP-RAG 容器端服务部署文档

本文档整合了 DP-RAG 所需的全部容器服务启动命令，并说明如何用 HuggingFace 镜像站下载模型、拉取所需镜像。

> 适用于 GPU 服务器（带 NVIDIA 显卡 + NVIDIA Container Toolkit）。

---

## 一、服务与端口总览

| 服务 | 容器名 | 镜像 | 端口 | 说明 |
|---|---|---|---|---|
| Milvus 向量库 | `milvus-standalone` | `milvusdb/milvus:v2.5.4` | 19530 / 9091 | 单机版（内嵌 etcd）；**数据落盘见下表** |
| Attu 可视化 | `milvus-attu` | `zilliz/attu:v2.5.10` | 3000 | Milvus 的 Web 管理界面 |
| 生成 LLM | `vllm-llm` | `vllm/vllm-openai:latest` | 8000 | Qwen3.5-9B，OpenAI 兼容接口 |
| Reranker | `vllm-rerank` | `vllm/vllm-openai:latest` | 8001 | Qwen3-Reranker-4B |
| Embedding | `vllm-embed` | `vllm/vllm-openai:latest` | 8002 | Qwen3-Embedding-4B |

**宿主机持久化路径（默认）**

| 用途 | 宿主机路径 | 容器内挂载点 | 环境变量 |
|---|---|---|---|
| Milvus 向量数据（etcd + 段文件） | `/fs_mol/wangzhengyan/dp-rag-milvus` | `/var/lib/milvus` | `MILVUS_DATA_DIR` |
| 模型权重（只读） | `<工作目录>/Qwen3-*` | `/models/Qwen3-*` | — |

推荐服务器目录布局：

```
/fs_mol/wangzhengyan/              # 工作目录 WORK_DIR (在此执行 docker run / compose)
  docker-compose.yml               # 可从 docs/docker-compose.yml 复制到此处
  Qwen3-Embedding-4B/
  Qwen3-Reranker-4B/
  Qwen3.5-9B/
  dp-rag-milvus/                   # Milvus 数据 (自动创建, 勿删)
  docs/start-services.sh           # 可选: 无 compose 时一键启动
```

**模型权重总量** ≈ 34GB（9B≈18GB + Reranker≈8GB + Embedding≈8GB）。

**显存占用**（按各命令的 `--gpu-memory-utilization`）：LLM 0.5 + Reranker 0.25 + Embedding 0.25，三者可共用同一张大显存卡（≥ 48GB 建议）。

### API Key（vLLM 鉴权）

三个 vLLM 服务共用同一个 API Key，通过启动参数 `--api-key` 设置。DP-RAG 后端通过环境变量 `VLLM_API_KEY` 调用（请求头 `Authorization: Bearer <key>`）。

```bash
# 启动容器前设置 (与后端 .env.local / default_config.yaml 保持一致)
export VLLM_API_KEY="dp_rag_vllm_sk_8f3a9c2e1b7d4f6a"
```

> 生产环境请自行更换为强随机密钥；`docker-compose.yml` 支持 `${VLLM_API_KEY:-默认值}` 覆盖。

### Milvus 数据目录

向量库数据（etcd + 段文件）默认持久化到宿主机：

```
/fs_mol/wangzhengyan/dp-rag-milvus
```

**首次部署前创建目录**（compose / docker run 启动 Milvus 前执行）：

```bash
sudo mkdir -p /fs_mol/wangzhengyan/dp-rag-milvus
sudo chown -R "$(id -u):$(id -g)" /fs_mol/wangzhengyan/dp-rag-milvus
# 若容器内 Milvus 无写权限, 可放宽: sudo chmod 777 /fs_mol/wangzhengyan/dp-rag-milvus
```

自定义路径时设置环境变量（与 `docker-compose.yml` 中 `${MILVUS_DATA_DIR}` 对应）：

```bash
export MILVUS_DATA_DIR=/your/custom/path
```

## 二、前置条件

```bash
# 1) 检查 Docker（>= 24.0）
docker --version

# 2) 检查 NVIDIA Container Toolkit（容器内能看到 GPU）
docker run --rm --gpus all nvidia/cuda:12.4.0-base-ubuntu22.04 nvidia-smi
```

若第 2 步报错，安装 NVIDIA Container Toolkit：

```bash
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo systemctl restart docker
```

## 三、拉取所需镜像

```bash
docker pull milvusdb/milvus:v2.5.4
docker pull zilliz/attu:v2.5.10
docker pull vllm/vllm-openai:latest
```

> 国内网络拉取镜像慢/失败时，可在 `/etc/docker/daemon.json` 配置镜像加速器后 `sudo systemctl restart docker`：
>
> ```json
> {
>   "registry-mirrors": ["https://docker.m.daocloud.io", "https://dockerproxy.com"]
> }
> ```
>
> 镜像加速器站点可能变动，若失效请换用其它可用的公共加速地址。

## 四、用 HF 镜像站下载模型

国内直连 `huggingface.co` 通常不可用，使用镜像站 [hf-mirror.com](https://hf-mirror.com)。

### 1. 安装下载工具

```bash
pip install -U "huggingface_hub[cli]"
```

### 2. 设置镜像端点（关键）

```bash
export HF_ENDPOINT=https://hf-mirror.com
# 如需长期生效, 写入 ~/.bashrc:
echo 'export HF_ENDPOINT=https://hf-mirror.com' >> ~/.bashrc && source ~/.bashrc
```

### 3. 下载三个模型到当前目录

下面的本地目录名要与各 `docker run` 命令里 `-v ./<目录>:/models/<目录>` 完全一致。

```bash
# Embedding（官方仓库: Qwen/Qwen3-Embedding-4B）
hf download Qwen/Qwen3-Embedding-4B --local-dir ./Qwen3-Embedding-4B

# Reranker（官方仓库: Qwen/Qwen3-Reranker-4B）
hf download Qwen/Qwen3-Reranker-4B --local-dir ./Qwen3-Reranker-4B

# 生成 LLM（按你的实际模型仓库填写; 目录名须为 Qwen3.5-9B）
hf download <你的LLM仓库ID> --local-dir ./Qwen3.5-9B
```

> ⚠️ 关于 LLM：你的命令里挂载的是 `./Qwen3.5-9B`。`Qwen3.5-9B` 不是 HuggingFace 上的标准公开仓库名，请把 `<你的LLM仓库ID>` 换成你实际使用的仓库（例如内部/私有仓库或对应的官方 Qwen 仓库）。只要**最终下载到的本地目录名是 `Qwen3.5-9B`**，后面的启动命令即可直接用。

**断点续传 / 重试**（大模型推荐）：

```bash
# huggingface-cli 自带断点续传, 网络中断后重跑同一条命令即可继续
# 如遇限速, 可加 --max-workers 调整并发
huggingface-cli download Qwen/Qwen3-Embedding-4B --local-dir ./Qwen3-Embedding-4B --max-workers 4
```

下载完成后确认目录结构（每个模型目录都应包含权重与配置）：

```bash
ls Qwen3-Embedding-4B/
# config.json  model-*.safetensors  tokenizer.json  ...
```

> 这些模型目录、以及后续启动命令，请都在**同一个工作目录**下执行（因为命令里用的是相对路径 `./模型名`）。
>
> Milvus 数据目录与模型目录**分开**：向量数据写入 `/fs_mol/wangzhengyan/dp-rag-milvus`，不要放在模型子目录里。

```bash
# 与第五节启动命令配套, 下载模型后执行一次
mkdir -p /fs_mol/wangzhengyan/dp-rag-milvus
```

## 五、启动各服务

有三种方式：**方式 C** 一键脚本（无 compose，下载慢时首选）、**方式 B** `docker compose`、**方式 A** 逐条 `docker run`。

### 方式 B：docker compose 一键启动（推荐）

本仓库提供了编排文件 `docs/docker-compose.yml`。

**工作目录要求**：必须在**同时包含**以下内容的目录下执行 compose（因为 `./Qwen3-*` 是相对路径）：

```
/fs_mol/wangzhengyan/          # 工作目录
  docker-compose.yml           # 从仓库 docs/docker-compose.yml 复制, 或 -f docs/docker-compose.yml
  Qwen3-Embedding-4B/
  Qwen3-Reranker-4B/
  Qwen3.5-9B/
  dp-rag-milvus/               # Milvus 数据 (启动前 mkdir -p)
```

先 `cd /fs_mol/wangzhengyan` 再启动（以下命令默认工作目录即此处）。

#### 先确认本机有 Compose

```bash
# Compose V2 插件 (推荐)
docker compose version

# 或 Compose V1 独立命令
docker-compose --version
```

若 `docker compose version` 报错 **unknown shorthand flag: 'f'**，说明未安装 V2 插件。按优先级尝试：

**① Compose V1（apt，多数云镜像可直接装）**

```bash
sudo apt-get update && sudo apt-get install -y docker-compose
docker-compose --version
# 之后用 docker-compose -f docs/docker-compose.yml up -d
```

**② Compose V2 插件（apt 找不到 docker-compose-plugin 时，手动安装；推荐）**

GitHub 直连慢时，用下方**加速地址**（任选其一）；加速站可能变动，失效就换另一条或走 scp。

```bash
COMPOSE_VER=v2.32.4
DEST=/usr/local/lib/docker/cli-plugins/docker-compose
sudo mkdir -p /usr/local/lib/docker/cli-plugins

# 方式 2a: GitHub 直连 (海外/带宽好时用)
# URL="https://github.com/docker/compose/releases/download/${COMPOSE_VER}/docker-compose-linux-x86_64"

# 方式 2b: ghproxy 加速 (国内常用)
URL="https://ghproxy.net/https://github.com/docker/compose/releases/download/${COMPOSE_VER}/docker-compose-linux-x86_64"

# 方式 2c: 备用加速
# URL="https://mirror.ghproxy.com/https://github.com/docker/compose/releases/download/${COMPOSE_VER}/docker-compose-linux-x86_64"

sudo curl -SL "$URL" -o "$DEST"
sudo chmod +x "$DEST"
docker compose version
```

**方式 2d: 本机下载后 scp 到服务器**（最稳）

```bash
# 在你自己的电脑上 (能较快访问 GitHub 或已挂代理)
curl -SL -o docker-compose "https://github.com/docker/compose/releases/download/v2.32.4/docker-compose-linux-x86_64"
chmod +x docker-compose
scp docker-compose root@<服务器IP>:/tmp/docker-compose

# 在服务器上
sudo mv /tmp/docker-compose /usr/local/lib/docker/cli-plugins/docker-compose
sudo chmod +x /usr/local/lib/docker/cli-plugins/docker-compose
docker compose version
```

**常见报错：`Not supported URL scheme http+docker`**

这是 **docker-compose V1 (Python 1.29.x)** 与新版 `urllib3`/`requests` 不兼容，与 compose 文件无关。
- **推荐**：按上面 ② 安装 Compose V2 二进制，用 `docker compose`（空格，无连字符）
- **勿再使用** `/usr/bin/docker-compose`（V1）除非降级 urllib3：`pip install 'urllib3<2'`

**③ 官方 apt 源安装插件**（仅当 Docker 本身来自 `download.docker.com` 时有效）

```bash
sudo apt-get update && sudo apt-get install -y docker-compose-plugin
```

#### 启动命令（二选一，语法相同）

```bash
cd /fs_mol/wangzhengyan
mkdir -p /fs_mol/wangzhengyan/dp-rag-milvus

# compose 文件在项目 docs/ 下, 或已复制为 ./docker-compose.yml
COMPOSE_FILE=docker-compose.yml   # 若用仓库路径则改为 docs/docker-compose.yml

# V2 插件
docker compose -f "$COMPOSE_FILE" up -d
docker compose -f "$COMPOSE_FILE" ps
docker compose -f "$COMPOSE_FILE" logs -f vllm-llm
docker compose -f "$COMPOSE_FILE" down    # 删容器, 保留 $MILVUS_DATA_DIR 内数据

# V1 独立命令 (未装插件时用这条)
docker-compose -f "$COMPOSE_FILE" up -d
docker-compose -f "$COMPOSE_FILE" ps
docker-compose -f "$COMPOSE_FILE" logs -f vllm-llm
docker-compose -f "$COMPOSE_FILE" down
```

> compose 与下方 docker run 均将 Milvus 数据固定挂载到 `/fs_mol/wangzhengyan/dp-rag-milvus`；删除容器不会丢失已灌入的向量数据。

### 方式 C：一键脚本（无需 compose，**下载慢时首选**）

不安装 docker compose，直接执行仓库脚本（内部是 `docker run`）：

```bash
cd /fs_mol/wangzhengyan
export MILVUS_DATA_DIR=/fs_mol/wangzhengyan/dp-rag-milvus
export WORK_DIR=/fs_mol/wangzhengyan
chmod +x docs/start-services.sh    # 脚本在仓库 docs/ 下; 或复制到当前目录
./docs/start-services.sh
```

等价于下方方式 A 的五条命令；容器已存在则跳过（重建需先 `docker rm -f <容器名>`）。

**若坚持用已有 docker-compose V1**，不必下载 60MB 的 V2 二进制，用清华 pip 源降级 urllib3 即可（几秒内完成）：

```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple 'urllib3<2' 'requests<2.32'
cd /fs_mol/wangzhengyan
export MILVUS_DATA_DIR=/fs_mol/wangzhengyan/dp-rag-milvus
mkdir -p "$MILVUS_DATA_DIR"
docker-compose -f docker-compose.yml up -d
```

### 方式 A：逐条 docker run

> 以下命令需在**包含三个模型目录的工作目录**下执行（`-v ./模型名` 是相对路径）。

### 1. Milvus 向量库

```bash
mkdir -p /fs_mol/wangzhengyan/dp-rag-milvus

docker run -d \
  --name milvus-standalone \
  --restart unless-stopped \
  --security-opt seccomp:unconfined \
  -e ETCD_USE_EMBED=true \
  -e ETCD_DATA_DIR=/var/lib/milvus/etcd \
  -e COMMON_STORAGETYPE=local \
  -v "/fs_mol/wangzhengyan/dp-rag-milvus:/var/lib/milvus" \
  -p 19530:19530 \
  -p 9091:9091 \
  milvusdb/milvus:v2.5.4 \
  milvus run standalone
```

### 2. Embedding（端口 8002）

```bash
docker run -d --gpus all \
  --name vllm-embed \
  -p 8002:8002 \
  --log-opt max-size=50m --log-opt max-file=3 \
  -v ./Qwen3-Embedding-4B:/models/Qwen3-Embedding-4B \
  vllm/vllm-openai:latest \
  /models/Qwen3-Embedding-4B \
  --host 0.0.0.0 --port 8002 \
  --api-key "${VLLM_API_KEY:-dp_rag_vllm_sk_8f3a9c2e1b7d4f6a}" \
  --trust-remote-code \
  --gpu-memory-utilization 0.25 \
  --max-model-len 9216
```

### 3. Reranker（端口 8001）

```bash
docker run -d --gpus all \
  --name vllm-rerank \
  -p 8001:8001 \
  --log-opt max-size=50m --log-opt max-file=3 \
  -v ./Qwen3-Reranker-4B:/models/Qwen3-Reranker-4B \
  vllm/vllm-openai:latest \
  /models/Qwen3-Reranker-4B \
  --host 0.0.0.0 --port 8001 \
  --api-key "${VLLM_API_KEY:-dp_rag_vllm_sk_8f3a9c2e1b7d4f6a}" \
  --trust-remote-code \
  --gpu-memory-utilization 0.25 \
  --max-model-len 10240 \
  --hf_overrides '{"architectures": ["Qwen3ForSequenceClassification"], "classifier_from_token": ["no", "yes"], "is_original_qwen3_reranker": true}'
```

### 4. 生成 LLM（端口 8000）

```bash
docker run -d --gpus all \
  --name vllm-llm \
  -p 8000:8000 \
  --log-opt max-size=50m --log-opt max-file=3 \
  -v ./Qwen3.5-9B:/models/Qwen3.5-9B \
  vllm/vllm-openai:latest \
  --model /models/Qwen3.5-9B \
  --host 0.0.0.0 --port 8000 \
  --api-key "${VLLM_API_KEY:-dp_rag_vllm_sk_8f3a9c2e1b7d4f6a}" \
  --max-model-len 30960 \
  --gpu-memory-utilization 0.5 \
  --max-num-seqs 128 \
  --reasoning-parser qwen3 \
  --enable-auto-tool-choice \
  --tool-call-parser hermes
```

### 5. Attu（Milvus 可视化，端口 3000）

```bash
docker run -d \
  --name milvus-attu \
  -p 3000:3000 \
  --restart unless-stopped \
  zilliz/attu:v2.5.10
```

## 六、启动后验证

```bash
# 查看全部容器状态
docker ps

# Milvus 健康检查
curl http://localhost:9091/healthz

# vLLM 三个服务的模型列表（确认已加载; 需带 API Key）
curl -H "Authorization: Bearer ${VLLM_API_KEY:-dp_rag_vllm_sk_8f3a9c2e1b7d4f6a}" http://localhost:8000/v1/models   # LLM
curl -H "Authorization: Bearer ${VLLM_API_KEY:-dp_rag_vllm_sk_8f3a9c2e1b7d4f6a}" http://localhost:8001/v1/models   # Reranker
curl -H "Authorization: Bearer ${VLLM_API_KEY:-dp_rag_vllm_sk_8f3a9c2e1b7d4f6a}" http://localhost:8002/v1/models   # Embedding
```

- vLLM 容器**首次启动需要加载权重**，大模型可能要等几十秒到几分钟；`curl /v1/models` 返回模型 ID 即就绪。
- 未带正确 `Authorization` 头时 vLLM 会返回 `401 Unauthorized`。
- Attu 打开浏览器访问 `http://<服务器IP>:3000`，连接地址填 `<服务器IP>:19530`。

## 八、与后端服务的对应关系

DP-RAG 后端（FastAPI，默认端口 8080）通过配置连接以上服务，默认地址：

| 后端配置项 | 指向 |
|---|---|
| `milvus` | `localhost:19530`（连接地址；**数据文件**在宿主机 `/fs_mol/wangzhengyan/dp-rag-milvus`） |
| `MILVUS_DATA_DIR`（宿主机） | `/fs_mol/wangzhengyan/dp-rag-milvus`（仅部署/运维用，后端配置里不写路径） |
| `generation`（LLM） | `http://localhost:8000/v1` |
| `reranker` | `http://localhost:8001/v1` |
| `embedding` | `http://localhost:8002/v1` |
| `VLLM_API_KEY`（环境变量） | 与 vLLM `--api-key` 一致，供后端 Bearer 鉴权 |

本地联调时在项目根 `source .env.local`（含 `VLLM_API_KEY`），再启动后端。

若后端与这些容器不在同一台机器，把 `localhost` 换成容器所在服务器的 IP（或用 SSH 端口转发）。
