# DP-RAG 容器端服务部署文档

本文档整合了 DP-RAG 所需的全部容器服务启动命令，并说明如何用 HuggingFace 镜像站下载模型、拉取所需镜像。

> 适用于 GPU 服务器（带 NVIDIA 显卡 + NVIDIA Container Toolkit）。

---

## 一、服务与端口总览

| 服务 | 容器名 | 镜像 | 端口 | 说明 |
|---|---|---|---|---|
| Milvus 向量库 | `milvus-standalone` | `milvusdb/milvus:v2.5.4` | 19530 / 9091 | 单机版（内嵌 etcd），存向量与元数据 |
| Attu 可视化 | `milvus-attu` | `zilliz/attu:v2.5.10` | 3000 | Milvus 的 Web 管理界面 |
| 生成 LLM | `vllm-llm` | `vllm/vllm-openai:latest` | 8000 | Qwen3.5-9B，OpenAI 兼容接口 |
| Reranker | `vllm-rerank` | `vllm/vllm-openai:latest` | 8001 | Qwen3-Reranker-4B |
| Embedding | `vllm-embed` | `vllm/vllm-openai:latest` | 8002 | Qwen3-Embedding-4B |

**模型权重总量** ≈ 34GB（9B≈18GB + Reranker≈8GB + Embedding≈8GB）。

**显存占用**（按各命令的 `--gpu-memory-utilization`）：LLM 0.5 + Reranker 0.25 + Embedding 0.25，三者可共用同一张大显存卡（≥ 48GB 建议）。

## 二、前置条件

```bash
# 1) 检查 Docker（>= 24.0）
docker --version

# 2) 检查 NVIDIA Container Toolkit（容器内能看到 GPU）
docker run --rm --gpus all nvidia/cuda:12.4.0-base-ubuntu22.04 nvidia-smi
```

若第 2 步报错，安装 NVIDIA Container Toolkit：

```bash
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo systemctl restart docker
```

## 三、拉取所需镜像

```bash
docker pull milvusdb/milvus:v2.5.4
docker pull zilliz/attu:v2.5.10
docker pull vllm/vllm-openai:latest
```

> 国内网络拉取镜像慢/失败时，可在 `/etc/docker/daemon.json` 配置镜像加速器后 `sudo systemctl restart docker`：
>
> ```json
> {
>   "registry-mirrors": ["https://docker.m.daocloud.io", "https://dockerproxy.com"]
> }
> ```
>
> 镜像加速器站点可能变动，若失效请换用其它可用的公共加速地址。

## 四、用 HF 镜像站下载模型

国内直连 `huggingface.co` 通常不可用，使用镜像站 [hf-mirror.com](https://hf-mirror.com)。

### 1. 安装下载工具

```bash
pip install -U "huggingface_hub[cli]"
```

### 2. 设置镜像端点（关键）

```bash
export HF_ENDPOINT=https://hf-mirror.com
# 如需长期生效, 写入 ~/.bashrc:
echo 'export HF_ENDPOINT=https://hf-mirror.com' >> ~/.bashrc && source ~/.bashrc
```

### 3. 下载三个模型到当前目录

下面的本地目录名要与各 `docker run` 命令里 `-v ./<目录>:/models/<目录>` 完全一致。

```bash
# Embedding（官方仓库: Qwen/Qwen3-Embedding-4B）
huggingface-cli download Qwen/Qwen3-Embedding-4B --local-dir ./Qwen3-Embedding-4B

# Reranker（官方仓库: Qwen/Qwen3-Reranker-4B）
huggingface-cli download Qwen/Qwen3-Reranker-4B --local-dir ./Qwen3-Reranker-4B

# 生成 LLM（按你的实际模型仓库填写; 目录名须为 Qwen3.5-9B）
huggingface-cli download <你的LLM仓库ID> --local-dir ./Qwen3.5-9B
```

> ⚠️ 关于 LLM：你的命令里挂载的是 `./Qwen3.5-9B`。`Qwen3.5-9B` 不是 HuggingFace 上的标准公开仓库名，请把 `<你的LLM仓库ID>` 换成你实际使用的仓库（例如内部/私有仓库或对应的官方 Qwen 仓库）。只要**最终下载到的本地目录名是 `Qwen3.5-9B`**，后面的启动命令即可直接用。

**断点续传 / 重试**（大模型推荐）：

```bash
# huggingface-cli 自带断点续传, 网络中断后重跑同一条命令即可继续
# 如遇限速, 可加 --max-workers 调整并发
huggingface-cli download Qwen/Qwen3-Embedding-4B --local-dir ./Qwen3-Embedding-4B --max-workers 4
```

下载完成后确认目录结构（每个模型目录都应包含权重与配置）：

```bash
ls Qwen3-Embedding-4B/
# config.json  model-*.safetensors  tokenizer.json  ...
```

> 这些模型目录、以及后续启动命令，请都在**同一个工作目录**下执行（因为命令里用的是相对路径 `./模型名`）。

## 五、启动各服务

有两种方式：**方式 A** 逐条 `docker run`（见下文），**方式 B** 用 `docker compose` 一键启动（推荐）。

### 方式 B：docker compose 一键启动（推荐）

本仓库提供了与下方命令等价的编排文件 `docs/docker-compose.yml`。把它和三个模型目录放在同一目录下，然后：

```bash
docker compose -f docker-compose.yml up -d     # 一键拉起全部 5 个服务
docker compose -f docker-compose.yml ps        # 查看状态
docker compose -f docker-compose.yml logs -f vllm-llm   # 跟日志
docker compose -f docker-compose.yml down      # 停止并删除容器(保留 Milvus 数据卷)
```

> compose 文件相比原始命令做了一处增强：给 Milvus 加了命名卷 `milvus-data` 做数据持久化（原始 `docker run` 未挂卷，数据会随容器删除而丢失）。

### 方式 A：逐条 docker run

> 以下命令需在**包含三个模型目录的工作目录**下执行（`-v ./模型名` 是相对路径）。

### 1. Milvus 向量库

```bash
docker run -d \
  --name milvus-standalone \
  --restart unless-stopped \
  --security-opt seccomp:unconfined \
  -e ETCD_USE_EMBED=true \
  -e ETCD_DATA_DIR=/var/lib/milvus/etcd \
  -e COMMON_STORAGETYPE=local \
  -p 19530:19530 \
  -p 9091:9091 \
  milvusdb/milvus:v2.5.4 \
  milvus run standalone
```

### 2. Embedding（端口 8002）

```bash
docker run -d --gpus all \
  --name vllm-embed \
  -p 8002:8002 \
  --log-opt max-size=50m --log-opt max-file=3 \
  -v ./Qwen3-Embedding-4B:/models/Qwen3-Embedding-4B \
  vllm/vllm-openai:latest \
  /models/Qwen3-Embedding-4B \
  --host 0.0.0.0 --port 8002 \
  --trust-remote-code \
  --gpu-memory-utilization 0.25 \
  --max-model-len 9216
```

### 3. Reranker（端口 8001）

```bash
docker run -d --gpus all \
  --name vllm-rerank \
  -p 8001:8001 \
  --log-opt max-size=50m --log-opt max-file=3 \
  -v ./Qwen3-Reranker-4B:/models/Qwen3-Reranker-4B \
  vllm/vllm-openai:latest \
  /models/Qwen3-Reranker-4B \
  --host 0.0.0.0 --port 8001 \
  --trust-remote-code \
  --gpu-memory-utilization 0.25 \
  --max-model-len 10240 \
  --hf_overrides '{"architectures": ["Qwen3ForSequenceClassification"], "classifier_from_token": ["no", "yes"], "is_original_qwen3_reranker": true}'
```

### 4. 生成 LLM（端口 8000）

```bash
docker run -d --gpus all \
  --name vllm-llm \
  -p 8000:8000 \
  --log-opt max-size=50m --log-opt max-file=3 \
  -v ./Qwen3.5-9B:/models/Qwen3.5-9B \
  vllm/vllm-openai:latest \
  --model /models/Qwen3.5-9B \
  --host 0.0.0.0 --port 8000 \
  --max-model-len 30960 \
  --gpu-memory-utilization 0.5 \
  --max-num-seqs 128 \
  --reasoning-parser qwen3 \
  --enable-auto-tool-choice \
  --tool-call-parser hermes
```

### 5. Attu（Milvus 可视化，端口 3000）

```bash
docker run -d \
  --name milvus-attu \
  -p 3000:3000 \
  --restart unless-stopped \
  zilliz/attu:v2.5.10
```

## 六、启动后验证

```bash
# 查看全部容器状态
docker ps

# Milvus 健康检查
curl http://localhost:9091/healthz

# vLLM 三个服务的模型列表（确认已加载）
curl http://localhost:8000/v1/models   # LLM
curl http://localhost:8001/v1/models   # Reranker
curl http://localhost:8002/v1/models   # Embedding
```

- vLLM 容器**首次启动需要加载权重**，大模型可能要等几十秒到几分钟；`curl /v1/models` 返回模型 ID 即就绪。
- Attu 打开浏览器访问 `http://<服务器IP>:3000`，连接地址填 `<服务器IP>:19530`。

## 七、常用运维命令

```bash
# 实时查看某服务日志（确认模型加载/排错）
docker logs -f vllm-llm

# 停止 / 启动 / 重启
docker stop  vllm-llm vllm-rerank vllm-embed milvus-standalone milvus-attu
docker start vllm-llm vllm-rerank vllm-embed milvus-standalone milvus-attu
docker restart vllm-llm

# 删除容器（不删除已下载模型 / Milvus 数据卷）
docker rm -f vllm-llm

# 查看 GPU 占用
nvidia-smi
```

## 八、与后端服务的对应关系

DP-RAG 后端（FastAPI，默认端口 8080）通过配置连接以上服务，默认地址：

| 后端配置项 | 指向 |
|---|---|
| `milvus` | `localhost:19530` |
| `generation`（LLM） | `http://localhost:8000/v1` |
| `reranker` | `http://localhost:8001/v1` |
| `embedding` | `http://localhost:8002/v1` |

若后端与这些容器不在同一台机器，把 `localhost` 换成容器所在服务器的 IP（或用 SSH 端口转发）。
